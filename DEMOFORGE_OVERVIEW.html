Step 1 — Ingest

3-step wizard captures project name, tagline, description, source type, screenshots (up to 6), URL, and style preset. Screenshots upload to Supabase Storage immediately.

Step 2 — Analyze + Outline

Optional: GitHub repo tree traversal scores and ranks files by relevance, then Claude extracts the tech stack, features, AI usage, and routes. Claude then generates a full scene-by-scene outline.

Step 3 — Render

Remotion bundles the composition, renders each scene as animated frames, and outputs an MP4 to /tmp/. The file is then uploaded to Supabase Storage and the URL saved to the DB.

Architecture

Frontend (Next.js App Router)

Server Components fetch data directly from Prisma. Client Components handle all interactive state (wizard, outline editor, polling).

Server Dashboard, demo detail page, Nav

Client Wizard, OutlinePanel, RepoPanel, RenderPanel, DemoPlayer

DemoPlayer is dynamically imported with ssr: false — Remotion's player uses browser APIs unavailable during SSR.

Backend (API Routes)

All mutations go through typed API routes. Each route verifies Supabase auth and demo ownership before operating.

Prisma 7 PrismaPg adapter with connection pooling via Supabase Transaction pooler (port 6543)

Render Background void runRender() task detached from HTTP response — client polls status endpoint

API Routes

Claude AI Integration

Call 1 — Repo Analysis

Route: POST /api/demos/[id]/analyze-repo

Input: Top ~15 files scored by relevance heuristic (route files +10, API dirs +8, AI hints +12, config +6). Each file truncated to 3,000 chars.

Output: CodeSummary — stack (frontend/backend/DBs/AI providers), features[], aiUsage[], routes[]

Validation: Zod schema with regex JSON extraction from Claude response.

claude-sonnet-4-6 max_tokens: 1500

Call 2 — Outline Generation

Route: POST /api/demos/[id]/generate-config

Input: projectName, tagline, description, stylePreset, screenshot count, features[], and codeSummary (if available)

Output: DemoConfig — theme, aspectRatio, scenes[] with type/headline/body/durationSeconds/cta/technicalNote

Validation: Zod schema enforcing scene structure. Sets renderStatus = "config_generated".

claude-sonnet-4-6 max_tokens: 2000

Remotion Video Engine

Each scene type maps to a Remotion <Sequence> with its own animation logic. durationSeconds × FPS(30) = frame count per scene.

Intro Scene

  • Title fade-up + scale entrance
  • Accent underline scaleX reveal
  • Tagline fade-in
  • Staggered bullet points
  • Radial gradient glow background
  • Bottom accent bar

Feature Scene

  • Screenshot with Ken Burns zoom + pan
  • Left panel slides in from -20px
  • Headline with opacity + translateY
  • Staggered bullet entrance
  • Tech stack badge
  • Left-to-right gradient blend overlay

Outro Scene

  • CTA scale-in entrance
  • Headline + body copy
  • technicalNote split into chips
  • Staggered chip entrance
  • Pulsing glow background
Theme tokens — clean (white/indigo), cyber (black/cyan), playful (dark/pink) — each defines bg, accent, gradient, and gridColor used across all three scenes.

Render Pipeline

POST /api/demos/[id]/render
  ├─ renderStatus = "rendering"  →  return 202
  └─ void runRender()                            // detached — never awaited
       ├─ const { bundle }           = await import("@remotion/bundler")
       ├─ const { renderMedia, ... } = await import("@remotion/renderer")
       ├─ bundleLocation = await bundle(remotionIndex, { webpackOverride })
       ├─ composition   = await selectComposition(bundleLocation, "DemoVideo", inputProps)
       ├─ await renderMedia({ composition, outputLocation: "/tmp/demoforge-{id}.mp4" })
       ├─ await supabaseAdmin.storage.upload("videos/{userId}/{demoId}/demo.mp4", file)
       └─ prisma.update({ renderStatus: "ready", videoUrl })

Key Engineering Decisions

Background Render Task

void runRender() is called without await so the HTTP response returns a 202 immediately. The client polls /render/status every 2 seconds. This works on long-running server environments (Railway, Render, VPS) but will time out on Vercel serverless. A dedicated render worker queue is the production path.

Prisma 7 + PrismaPg Adapter

Prisma 7 removed url = env("DATABASE_URL") from schema.prisma — the URL now lives only in prisma.config.ts (for CLI) and the PrismaPg({ connectionString }) constructor (for runtime). @types/pg must not be installed separately — it conflicts with the version bundled inside @prisma/adapter-pg.

Remotion Webpack Externals

@remotion/bundler ships native .node binaries (@rspack/binding-darwin-arm64.node). Next.js webpack will crash trying to parse these. Both experimental.serverComponentsExternalPackages and webpack.externals must include all four Remotion packages.

Prisma JsonValue Casting

Prisma's JsonValue and custom TypeScript interfaces (e.g., DemoConfig) don't overlap, so direct casts like raw.demoConfig as DemoConfig fail. The pattern used throughout is a double cast: raw.demoConfig as unknown as DemoConfig.

Supabase Admin Client for Storage

RLS policies on Supabase Storage buckets would block server-side uploads from an API route. createAdminClient() uses SUPABASE_SERVICE_ROLE_KEY to bypass RLS entirely. This client is only instantiated server-side and never exposed to the browser.

GitHub Provider Token for Octokit

When a user signs in with GitHub OAuth, Supabase stores their GitHub access token in the session as provider_token. The analyze-repo route reads this token from the server-side session and passes it to Octokit, giving it the user's own rate-limit quota (5,000 req/hr) without any extra setup.

Deployment

Recommended: Railway / Render

DemoForge needs a persistent Node.js process for the background render task. Railway and Render both provide this with zero config.

  1. Push to GitHub
  2. Create a new Railway/Render service pointing at the repo
  3. Set all environment variables in the service dashboard
  4. Build command: npm run build
  5. Start command: node node_modules/next/dist/bin/next start

Vercel (partial)

The web app (dashboard, wizard, auth, outline editor) deploys fine to Vercel. The render step will time out on serverless functions.

To run on Vercel, move the render pipeline to a separate worker: a Railway service, a queue job (BullMQ), or a Remotion Lambda deployment.
